# ai-text-detector
A machine learning project that classifies whether a piece of text was written by a **human** or generated by an **AI** (e.g., ChatGPT). Built using handcrafted linguistic features and a Random Forest classifier — no heavy NLP libraries required.

---

## Problem Statement

With the rise of large language models, educators and institutions face a growing challenge: distinguishing AI-generated submissions from genuine student work. This project builds a lightweight, interpretable classifier that flags potentially AI-written text based on measurable linguistic patterns.

---

## How It Works

Instead of relying on a neural network, this detector extracts **15 handcrafted features** from raw text:

| # | Feature | Why it matters |
|---|---------|----------------|
| 1 | Average word length | AI uses more formal/longer vocabulary |
| 2 | Average sentence length | AI produces longer, more uniform sentences |
| 3 | Sentence length std. deviation | Humans show higher "burstiness" |
| 4 | Type-token ratio | Lexical diversity; AI can be repetitive |
| 5 | Punctuation ratio | AI follows stricter grammatical patterns |
| 6 | Comma ratio | AI over-uses commas in structured lists |
| 7 | Exclamation ratio | Humans use more emotional punctuation |
| 8 | Question ratio | Relative frequency of questions |
| 9 | Uppercase word ratio | Humans capitalise for emphasis/emotion |
| 10 | Digit ratio | AI inserts statistics more frequently |
| 11 | Avg paragraph length | AI paragraphs tend to be more uniform |
| 12 | Stopword ratio | Functional word density differs |
| 13 | Long word ratio | Words longer than 6 characters |
| 14 | Sentence count | Total sentences in the text |
| 15 | Whitespace ratio | Formatting artefacts |

These features are fed into a **Random Forest classifier** trained on a large public dataset of AI and human-written essays.

## Setup

### 1. Clone the repository
```bash
git clone https://github.com/YOUR_USERNAME/ai-text-detector.git
cd ai-text-detector
```

### 2. Install dependencies
```bash
pip install -r requirements.txt
```

### 3. Download the dataset
1. Go to: https://www.kaggle.com/datasets/shanegerami/ai-vs-human-text
2. Download `AI_Human.csv`
3. Place it inside the `data/` folder:
```
data/AI_Human.csv
```

---

## Usage

### Step 1 — Train the model
```bash
python train.py
```
This will:
- Load and balance 20,000 samples from the dataset
- Extract linguistic features
- Train a Random Forest classifier
- Print accuracy + classification report
- Save the model to `model/detector.pkl`

Expected output:
```
[1/4] Loading data from: data/AI_Human.csv
    Loaded 20000 samples  (human=10000, AI=10000)
[2/4] Extracting features …
    Feature matrix shape: (20000, 15)
[3/4] Training Random Forest …
[4/4] Evaluating …

    Accuracy : 0.8650

    Classification Report:
              precision    recall  f1-score   support
       Human       0.87      0.86      0.86      2000
          AI       0.86      0.87      0.87      2000
```

### Step 2 — Predict on new text

**Interactive mode** (recommended for testing):
```bash
python predict.py
```
Paste text, press Enter twice to get a prediction.

**Single string:**
```bash
python predict.py "The implications of climate change are multifaceted and require a comprehensive policy response."
```

**From a .txt file:**
```bash
python predict.py --file my_essay.txt
```

### Step 3 — Analyse features (optional)
```bash
python analyze.py
```
Generates a feature importance chart and a human-vs-AI comparison table.

## Limitations

- Works best on texts of **50+ words**; short texts reduce reliability
- Trained on essays; may behave differently on code, poetry, or casual chat
- AI writing styles evolve — a model trained today may need retraining over time
- This is a **probabilistic tool**, not ground truth; always apply human judgement
